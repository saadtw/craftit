Admin Portal 
1. Basic/Main Flow
1.1 Admin Login
•	Email input field
•	Password input field
•	Two-factor authentication 
•	Login button
1.2 Admin Dashboard (Main Hub)
•	Quick statistics cards: 
o	Pending manufacturer verifications
o	Active disputes 
o	Total users
o	Total manufacturers
o	Active orders count
•	Critical alerts panel (issues requiring immediate attention)
•	Quick action tiles: 
o	Verify manufacturers
o	Resolve disputes
o	View all users
o	View all orders
•	Navigation menu
•	Recent admin activities log (last 10 actions)
2. Manufacturer Verification 
2.1 Verification Queue
•	List of pending manufacturer applications
•	Application cards showing: 
o	Business name
o	Date submitted
o	Completeness indicator
o	Location
o	Quick review button
•	Filter by date submitted
•	Sort by date (oldest first priority)
2.2 Manufacturer Details
•	Business information display: 
o	Business name
o	Registration number
o	Tax ID
o	Contact person and details
o	Address
•	Documents viewer
•	Manufacturing capabilities review
•	Simple checklist: 
o	[ ] Business legitimacy verified
o	[ ] Documents appear authentic
o	[ ] Contact information verified
•	Action buttons: 
o	Approve (sets account to verified, sends welcome email)
o	Reject (requires reason, sends rejection email)
o	Request More Info (specify what's needed, sets 7-day deadline)
2.3 Approve Manufacturer
•	Confirmation dialog
•	Approve button
•	System automatically: 
o	Sets verification badge
o	Activates manufacturer account
o	Sends approval email
o	Grants full platform access
2.4 Reject Manufacturer
•	Rejection reason dropdown: 
o	Incomplete documents
o	Documents appear fraudulent
o	Cannot verify business
o	Other (specify)
•	Detailed feedback text area
•	Send rejection email checkbox (checked by default)
•	Confirm rejection button
3. User Management 
3.1 All Users List
•	Searchable list of all users (customers + manufacturers)
•	User cards showing: 
o	Name and email
o	User type (Customer/Manufacturer)
o	Registration date
o	Status (Active/Suspended)
•	Search by name or email
•	Filter by: 
o	User type
o	Status
o	Registration date range
3.2 User Details (View Only)
•	User profile information
•	Account statistics: 
o	Total orders (for customers)
o	Total revenue (for manufacturers)
o	Registration date
o	Last active
•	Order history (list)
•	Basic actions: 
o	Suspend account (requires reason)
o	Unsuspend account
o	Send email to user
3.3 Suspend User
•	Suspension reason selection: 
o	Terms violation
o	Multiple complaints
o	Fraudulent activity
o	Other (specify)
•	Detailed reason (required)
•	Suspension duration: 
o	7 days
o	30 days
o	Permanent
•	Notify user checkbox
•	Confirm suspension button
4. Order Oversight (Basic Monitoring)
4.1 All Orders List
•	List of all platform orders
•	Order cards showing: 
o	Order ID
o	Customer name
o	Manufacturer name
o	Status
o	Order value
o	Date placed
o	Issue flag (if any)
•	Search by order ID
•	Filter by: 
o	Status
o	Date range
o	Has disputes/issues
4.2 Order Details (View Only)
•	Complete order information
•	Customer and manufacturer details
•	Order timeline
•	Payment status
•	Any issues/disputes linked
•	View only - basic oversight
•	Contact customer/manufacturer buttons (email)

5. Dispute Resolution (Essential)
5.1 Disputes List
•	Active disputes requiring attention
•	Dispute cards showing: 
o	Dispute ID
o	Order reference
o	Customer name
o	Manufacturer name
o	Issue type
o	Date opened
o	Status (New/Under Review/Resolved)
•	Filter by status
•	Sort by date (oldest first)
5.2 Dispute Details
•	Dispute information: 
o	What happened (customer's claim)
o	Order details
o	Evidence uploaded (photos, documents)
•	Manufacturer's response (if provided)
•	Timeline of events
•	Admin notes section (internal only)
5.3 Resolve Dispute
•	Review all information
•	Decision options: 
o	Refund Customer (full or partial) 
	Refund amount input
o	Side with Manufacturer (no refund)
o	Partial Resolution (compromise)
•	Resolution message to both parties (required)
•	Finalize resolution button
•	Actions automatically executed: 
o	Process refund (if applicable)
o	Update order status
o	Close dispute
o	Notify both parties
6. Account 
6.1 Admin Profile
•	View admin details
•	Change password
•	Two-factor authentication settings
•	Activity log (own actions)
6.2 Admin Activity Log
•	List of all admin actions with: 
o	Action type
o	Details
o	Admin who performed it
o	Timestamp
•	Search and filter options
•	Maintains audit trail
