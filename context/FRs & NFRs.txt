Functional Requirements
1.	Authentication & User Management
1.1.	Allow registration for customers or manufacturers only.
1.2.	Enforce role-based access for customers, manufacturers, and admins.
1.3.	Manufacturers must submit business registration documents during signup.
1.4.	Allow users to login with email and password.
1.5.	Encrypt user credentials for secure storage.
2.	Customer Dashboard
2.1.	Customer Dashboard is to be accessible only after successful registration and login.
2.2.	Display account overview and stats on customer dashboard with recent notifications.
2.3.	Allow customers to browse manufacturers and products.
2.4.	Allow viewing of order history and current order status.
2.5.	Provide functionality for customers to add products to Wishlist.
3.	Manufacturer Dashboard
3.1.	Display metrics including total orders, revenue, and bids count for manufacturers.
3.2.	Display past and active orders with search and filters.
3.3.	Ensure the display of RFQs that matches manufacturer capabilities.
3.4.	Show all submitted bids with their status.
3.5.	Allow manufacturers to update order status through production milestones.
3.6.	Enable manufacturers to update order progress and milestones visible to customers.
4.	Product Catalog Management
4.1.	Allow manufacturers to create product listings with name, category, description, price, and MOQ.
4.2.	Support uploads of images and 3D models.
4.3.	Implement search across product names, descriptions, and categories with filtering by price, category, and MOQ.
5.	3D Model Viewer
5.1.	Provide 3D viewer supporting rotation, zoom, and pan.
5.2.	Process 3D models to be optimized for web viewing.
5.3.	Display the model dimensions automatically where possible.
6.	Order Processing & Tracking
6.1.	Allow customers to place orders from product listings.
6.2.	Authorize payment on order placement and capture after manufacturer acceptance.
6.3.	Allow customers to cancel order before manufacturer accepts, with full refund.
6.4.	Provide messaging capabilities between customers and manufacturers per order.
7.	Custom Order and RFQ System
7.1.	Allow customers to create custom order requests with title, description, quantity, budget, deadline, and file uploads.
7.2.	Allow customers to create RFQ auctions with configurable duration.
8.	RFQ & Bidding System
8.1.	Enable manufacturers to submit bids including amount, timeline, and additional notes.
8.2.	Allow customers to view and compare bids.
8.3.	Support counter-offer negotiation between customers and manufacturers.
8.4.	Automatically convert the accepted bids into separate orders for both customers and manufacturers.
8.5.	Close auctions automatically once a bid is accepted and notify participating manufacturers.
9.	Group Buy Management
9.1.	Allow manufacturers to create group buy campaigns with product selection, duration, and discount tiers.
9.2.	Validate tier logic ensuring quantity corresponds to discount percentages.
9.3.	Display group buy listings with real-time participant count, progress bar, and countdown timer.
9.4.	Authorize payments when customers join.
9.5.	Create individual orders for each participant upon campaign completion.
9.6.	Refund all participants if campaign is cancelled or minimum tier not reached.
10.	Payment Processing
10.1.	Integrate with payment gateway supporting credit/debit cards.
10.2.	Tokenize payment information and never store raw data.
10.3.	Hold manufacturer payments until order completion.
10.4.	Process refunds to original payment method incase of returns or refund resolution of disputes.
11.	Admin Management
11.1.	Provide admin dashboard displaying pending manufacturer verifications, active disputes, total users, and active orders.
11.2.	Display manufacturer verification queue with submitted documents.
11.3.	Allow admins to approve or reject manufacturer applications.
11.4.	Allow admins to view user details and suspend accounts with reason.
11.5.	Display all platform orders with search capabilities.
11.6.	Allow admins to view dispute details and apply decisions.
11.7.	Maintain audit logs of all admin actions.
 
Non Functional Requirements
1.	Performance:
1.1.	Achieve page loading under 4 seconds on standard connections.
1.2.	API response time must be less than 0.5 seconds.
2.	Scalability:
2.1.	Ensure that the system is able to handle 500 users at a time without an issue in the performance.
3.	Reliability
3.1.	Average downtime over a period of two months must not be above 1%
3.2.	Scheduled automated database backups
4.	Security
4.1.	Ensure only authorized users are able to access the system resources.
4.2.	Ensure multi-factor authentication is enforced on all accounts.
5.	Usability
5.1.	Average user shall be able to understand the use of the system in maximum of 2 hours.
5.2.	A responsive interface must be implemented for both, computer and mobile devices.
6.	Maintainability
6.1.	Provide complete documentation of all of the features for every type of user.
